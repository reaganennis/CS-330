# CS-330
When I design software, I like to break things down into small pieces and figure out what each part needs to do. For this project, that meant understanding how shapes, lighting, textures, and camera movement all fit together in a 3D scene. Working on this helped me build stronger spatial thinking skills and get more comfortable making design choices that affect how the final scene feels.

My design process was simple and iterative: start with basic shapes, add detail, test how it looks, and adjust. That step‑by‑step approach is something I can use in future projects, especially when dealing with visuals or complex systems

When I develop programs, I focus on building small parts first and testing as I go. In this project, I used new strategies like checking shader behavior early, isolating transformations, and constantly rendering the scene to see what needed fixing. Iteration played a huge role—I’d tweak something, run it, see how it looked, and repeat.

Across the milestones, I went from “just get objects on the screen” to understanding how the graphics pipeline works as a whole. By the end, I was thinking more intentionally about how each change would affect the final scene.


Computer science gives me tools to solve problems and build things that matter. Graphics and visualization add another layer, they teach me how to represent ideas visually and make environments or data easier to understand.

These skills help me academically by deepening my understanding of how systems work behind the scenes. Professionally, they open doors to fields like simulation, visualization, secure software development, and any job where understanding 3D environments or visual communication is useful.
